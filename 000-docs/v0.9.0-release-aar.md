## Lessons Learned

1. **Gemini PR review churn**: Round-2 suggestions that reverse round-1 feedback should be dismissed, not iterated. Documented in beads memory.

2. **Lazy snapshot diffing**: The `pairing.accepted` design (detect state changes in existing hot path rather than introducing IPC) is reusable for future cross-process observability needs.

3. **Complexity extraction pays off**: The refactor pass (ccsc-53g umbrella) dropped the worst function from 84 → 5. Investment in extraction helpers yields compounding returns on future changes.
